# 第五章 异常与文件复习提纲

本章重点掌握异常处理机制（try-except-else-finally结构）与文件操作规范（with语句、编码指定、读写模式），结合实验要求理解错误处理与数据持久化实践。

## 异常处理基础

- 使用try包裹可能失败的代码，配合except捕获特定异常类型
- else块在无异常时执行，finally块无论是否异常均会执行
- 应捕获具体异常类型，避免无条件吞掉所有异常

**易错点**

- 使用裸except捕获所有异常导致错误被隐藏
- 混淆else与finally的执行条件

## 文件操作规范

- 使用with open(path, mode, encoding='utf-8')确保文件自动关闭
- 常见模式：r（读取）、w（覆盖写入）、a（追加写入）
- 处理文本文件必须明确指定编码格式

**易错点**

- 忘记指定encoding参数导致编码错误
- 未使用with语句造成文件资源泄漏

## 综合应用示例

- 文件不存在时捕获FileNotFoundError并提示用户
- 读取文本后统一处理数据（如转小写）再统计
- 结果排序需满足多条件规则（频率降序+字母顺序）

**易错点**

- 未处理文件路径错误导致程序崩溃
- 排序逻辑未覆盖所有指定条件
